Project Link
  • Tech Stack: Python, Apache Spark, Airflow, SQL, ETL, Amazon Web Services (AWS)

GoodReads Data Pipeline Project Description The GoodReads Data Pipeline is an end-to-end data processing and analytics solution designed to capture, transform, and store data from the Goodreads API in real time. The pipeline is divided into several key modules, including a Goodreads Python Wrapper, ETL Jobs, a Redshift Warehouse Module, and an Analytics Module. Below is an overview of how the pipeline operates:

Overview Data Capture: Data is extracted in real time using the Goodreads Python Wrapper from the Goodreads API. The collected data is initially stored on local disk and then periodically moved to an AWS S3 Landing Bucket. ETL Jobs: The pipelineâ€™s ETL (Extract, Transform, Load) jobs, written in Apache Spark, are scheduled using Airflow to run every 10 minutes, ensuring that data is continuously processed and updated.

ETL Flow Landing Zone: Data from the Goodreads API is moved to an S3 bucket (Landing Zone). Working Zone: The ETL job's S3 module transfers data from the Landing Zone to the Working Zone. Data Transformation: A Spark job is triggered, reading the data from the Working Zone and applying necessary transformations. The dataset is then repartitioned and moved to the Processed Zone on S3. Data Warehouse Loading: The Redshift Warehouse Module loads data from the Processed Zone into Redshift staging tables. Using UPSERT operations, the data in the staging tables is used to update the final tables in the Data Warehouse. ETL Job Completion: The ETL job completes once the Data Warehouse is updated with the processed data.

Airflow DAG & Analytics Once the ETL job is complete, Airflow DAG triggers a data quality check on all the Data Warehouse tables to ensure data integrity. The pipeline also includes an Analytics Module, where custom Airflow Operators run predefined analytics queries. A final data quality check is performed on select analytics tables before DAG execution is marked complete.

This pipeline ensures reliable, real-time data ingestion, transformation, and analytics processing, leveraging AWS infrastructure, Spark, and Airflow for efficient data operations.
